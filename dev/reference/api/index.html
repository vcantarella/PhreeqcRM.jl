Lifecycle

PhreeqcRM.PhreeqcRMInstance — Type
PhreeqcRMInstance(nxyz; nthreads=1, component_h2o=false)

Construct a PhreeqcRM instance covering nxyz reaction cells. Returns an opaque handle around an integer id and a small amount of Julia-side bookkeeping (no C-owned memory references).

Keyword arguments:

  • nthreads::Integer=1 — OpenMP thread count for the internal RunCells loop. The default is 1 to avoid OpenMP × Julia thread oversubscription. Set higher (or Threads.nthreads()) when reactions dominate runtime.
  • component_h2o::Bool=false — if true, water is tracked as its own component instead of being inferred from H and O moles. Affects the component list reported by find_components!. Must be set before load_database! — that's why it lives on the constructor.

The constructor does not create any output files. Use open_output! to enable *.chem.txt and *.log.txt logs.

Always pair with Base.close or use with_instance.

PhreeqcRM.with_instance — Function
with_instance(f, nxyz; kwargs...)

Construct a PhreeqcRMInstance(nxyz; kwargs...), pass it to f, and close it on exit — including when f throws.

with_instance(40; nthreads=4) do rm
    load_database!(rm, "phreeqc.dat")
    run_file!(rm, "advect.pqi")
    # ...
end
Base.close — Method
close(rm::PhreeqcRMInstance)

Destroy the PhreeqcRM instance and release its OpenMP workers. Idempotent.

PhreeqcRM.open_output! — Function
open_output!(rm; prefix, dir=pwd())

Open PhreeqcRM's text + log output files at joinpath(dir, prefix).chem.txt and .log.txt. Opt-in — the constructor does not create files.

PhreeqcRM.close_output! — Function
close_output!(rm)

Close any output files previously opened with open_output!.

Database and PHREEQC input

PhreeqcRM.load_database! — Function
load_database!(rm, path::AbstractString)

Read a PHREEQC database file (e.g. phreeqc.dat, llnl.dat, pitzer.dat) into all the IPhreeqc instances inside rm. Must precede run_file!.

PhreeqcRM.run_file! — Function
run_file!(rm, path; workers=true, initial=true, utility=true)

Run a PHREEQC input file against the selected IPhreeqc instances inside rm.

  • workers=true — populate the worker IPhreeqc instances (the ones that do per-cell chemistry inside run_cells!).
  • initial=true — populate the InitialPhreeqc instance, which holds the numbered SOLUTION / EXCHANGE / EQUILIBRIUM_PHASES definitions later referenced by set_initial_conditions!.
  • utility=true — populate the Utility instance.

The defaults match the common case: a PHREEQC script that defines the full chemistry. Call once before find_components!.

PhreeqcRM.run_string! — Function
run_string!(rm, input; workers=true, initial=true, utility=true)

Same as run_file! but takes a PHREEQC input as a string. Useful for programmatically generated scripts or inline test fixtures.

Components

PhreeqcRM.find_components! — Function
find_components!(rm) -> Vector{String}

Scan the InitialPhreeqc instance for every element / mass-balance component that has been defined by run_file! / run_string!, and cache the result on rm.

Must be called before any set_concentrations! / get_concentrations! call because the concentration buffer is sized ncomps(rm) * nxyz(rm). Subsequent calls are cheap (results are cached).

PhreeqcRM.components — Function
components(rm) -> Vector{String}

Cached component names from the most recent find_components!. Throws if find_components! hasn't been called yet.

PhreeqcRM.ncomps — Function
ncomps(rm) -> Int

Number of components in rm's chemistry. Zero until find_components! is called.

PhreeqcRM.nxyz — Function
nxyz(rm) -> Int

Number of reaction cells in rm (the value passed to the constructor).

Units

PhreeqcRM.set_units! — Function
set_units!(rm; solution, pp_assemblage=..., exchange=..., surface=...,
           gas_phase=..., ss_assemblage=..., kinetics=...)

Configure the unit convention for each reactant category. solution is required — its value determines how the numbers passed to set_concentrations! are interpreted (mg/L vs mol/L vs kg/kgs).

All *_assemblage / exchange / surface / gas_phase / kinetics units default to mol/L of rock volume.

Each keyword takes a value from the matching enum: SolutionUnits.MolPerL, PPAssemblageUnits.MolPerLRock, etc.

(See also the per-domain enum modules: SolutionUnits, PPAssemblageUnits, ExchangeUnits, SurfaceUnits, GasPhaseUnits, SSAssemblageUnits, KineticsUnits.)

Spatial properties

PhreeqcRM.set_mapping! — Function
set_mapping!(rm, grid2chem::AbstractVector{<:Integer})

Map each of nxyz transport cells to a reaction cell index. -1 marks an inactive transport cell that should not be reacted. Default mapping is identity (collect(0:nxyz-1)); call this only if you need to skip cells or collapse multiple transport cells onto one reaction cell.

The per-cell scalar setters all share the same signature f(rm, v::AbstractVector{<:Real}) where length(v) == nxyz(rm). They send the per-cell physical property to PhreeqcRM's internal store:

  • set_porosity!(rm, v) — porosity (unitless)
  • set_saturation!(rm, v) — saturation (unitless)
  • set_representative_volume!(rm, v) — representative volume (L)
  • set_temperature!(rm, v) — temperature (°C)
  • set_pressure!(rm, v) — pressure (atm)
  • set_density!(rm, v) — density (kg/L)

Concentrations

PhreeqcRM.zeros_concentrations — Function
zeros_concentrations(rm) -> Matrix{Float64}

Allocate a fresh concentration matrix sized (nxyz(rm), ncomps(rm)), initialized to zero. This is the recommended constructor — passing the wrong shape to set_concentrations! is an error, but constructing this way makes the contract self-enforcing.

Each row holds one cell; each column holds one component across cells.

PhreeqcRM.set_concentrations! — Function
set_concentrations!(rm, c::AbstractMatrix{Float64})

Send the per-cell, per-component concentrations to PhreeqcRM. c must have shape (nxyz(rm), ncomps(rm)) and contiguous strides — the recommended way to allocate it is zeros_concentrations. Validated, zero-copy in the happy path.

PhreeqcRM.get_concentrations — Function
get_concentrations(rm) -> Matrix{Float64}

Allocating convenience that returns a fresh Matrix{Float64}(ncomps, nxyz) filled with current concentrations. Do not call inside a hot loop — use get_concentrations! with a pre-allocated buffer instead.

PhreeqcRM.get_concentrations! — Function
get_concentrations!(rm, c::AbstractMatrix{Float64})

Pull the post-reaction concentrations from PhreeqcRM into the caller's buffer c. Shape and contiguity requirements match set_concentrations!. In-place — does not allocate. This is what the hot loop should use.

PhreeqcRM.bycell — Function
bycell(c::AbstractMatrix{Float64})

Iterator over per-cell row views of a concentration matrix (nxyz, ncomps). Each element is a view (no copy) of length ncomps. Convenient for per-cell inspection:

for (icell, cell) in enumerate(bycell(c))
    @show icell, cell[1]   # first component in this cell
end

Initial / boundary conditions

PhreeqcRM.set_initial_conditions! — Function
set_initial_conditions!(rm; solution=fill(-1, nxyz(rm)),
                            equilibrium_phases=..., exchange=...,
                            surface=..., gas_phase=...,
                            ss_assemblage=..., kinetics=...)

Assign numbered reactant blocks (defined in the PHREEQC script previously loaded with run_file!) to each of the nxyz cells.

Each keyword takes an integer vector of length nxyz — entry i is the PHREEQC user number of the block to use in cell i, or -1 to leave that slot empty.

Unspecified categories default to -1 in every cell.

PhreeqcRM.initial_phreeqc_to_concentrations — Function
initial_phreeqc_to_concentrations(rm; solution::AbstractVector{<:Integer}) -> Matrix{Float64}

Translate a list of SOLUTION N definitions from the InitialPhreeqc instance into a concentration matrix of shape (length(solution), ncomps(rm)) — same layout convention as zeros_concentrations: each row is one boundary solution, each column is one component.

Typical use: build the inflow/boundary concentration vector(s) for transport without touching any cell:

bc = initial_phreeqc_to_concentrations(rm; solution=[0])   # SOLUTION 0, 1×ncomps

Time stepping

PhreeqcRM.set_time! — Function
set_time!(rm, t::Real)

Set the current simulation time (PhreeqcRM-internal seconds by default).

PhreeqcRM.set_time_step! — Function
set_time_step!(rm, dt::Real)

Set the time step over which the next run_cells! call will integrate kinetic reactants. Must be called before run_cells! or kinetics integrate over zero time.

PhreeqcRM.run_cells! — Function
run_cells!(rm)

Run the reaction calculation on every active cell, in parallel across PhreeqcRM's OpenMP workers. The current cell concentrations are read from the buffer last seen by set_concentrations!; the post-reaction concentrations are stored in PhreeqcRM's internal buffer, retrievable with get_concentrations!.

Selected output

PhreeqcRM.enable_selected_output! — Function
enable_selected_output!(rm, on::Bool)

Turn the recording of SELECTED_OUTPUT columns on or off. Defaults to off in the upstream library. Call before run_cells! to get fresh values out of get_selected_output.

PhreeqcRM.get_selected_output — Function
get_selected_output(rm) -> NamedTuple

Return the current SELECTED_OUTPUT table as a NamedTuple whose names are the column headings (as Symbols) and whose values are Vector{Float64} of length RM_GetSelectedOutputRowCount(rm) (which equals nxyz when the selected output is per cell).

Allocates. Use get_selected_output! for hot paths.

PhreeqcRM.selected_output_headings — Function
selected_output_headings(rm) -> Vector{String}

Column names from the SELECTED_OUTPUT block. Cached on first call.

PhreeqcRM.get_solution_volume — Function
get_solution_volume(rm) -> Vector{Float64}

Per-cell solution volume (L) after the most recent run_cells!.

PhreeqcRM.get_density — Function
get_density(rm) -> Vector{Float64}

Per-cell solution density (kg/L) computed from the most recent run_cells!.

PhreeqcRM.get_saturation — Function
get_saturation(rm) -> Vector{Float64}

Per-cell saturation (unitless) computed from the most recent run_cells! — may differ from the value last passed to set_saturation! if mineral dissolution / precipitation changed the solution volume.
